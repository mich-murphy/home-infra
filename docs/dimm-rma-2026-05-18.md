## Failure summary

Deterministic, reproducible hardware fault. MemTest86 hangs at **Test 1 (own-address test), ~70% completion**, while sweeping the physical address region around **`0x800000000`–`0x87F80D000`**. Slot-independent. Reseating allows POST but does not eliminate the hang.

## Test matrix

| Test | Slot | Module | Result |
|------|------|--------|--------|
| 1 | DIMMA2 + DIMMB2 (dual) | Both populated | MemTest hangs Test 1 @ 74%, addr region `0x107F80D000` |
| 2 | DIMMB2 only | Good (`...4D5A02A7`) | **4 full passes clean, all 13 tests, 0 errors** |
| 3 | DIMMA2 only | Faulty (`...4D9A8E3E`) | MemTest hangs Test 1 @ 70%, addr region `0x87F80D000` |
| 4 | DIMMB2 only | Faulty (`...4D9A8E3E`) | Intermittent POST failure (5 short + 1 long AMI beep = memory training failure). When POST succeeded, MemTest hung Test 1 @ 70%, addr region `0x87F80D000` |

The failure follows the **module across slots**. The control DIMM (same part, same system, same slot) passes cleanly. Rules out motherboard slot, CPU IMC, BIOS, companion module as the fault source.

## Production impact

Hard crash on **2026-05-18 11:32 AEST**. Linux kernel reported an ACPI BERT record with FATAL severity on the subsequent boot:

```
BERT: [Hardware Error]: Skipped 1 error records
BERT: Total records found: 1
sysfs: cannot create duplicate filename '/firmware/acpi/tables/data/BERT'
 acpi_bert_data_init+0x37/0x60
```

Raw BERT payload (`/sys/firmware/acpi/tables/data/BERT`, 17912 bytes):

- Generic Error Status Block: ErrorSeverity = `01` (FATAL)
- Generic Error Data Entry SectionType GUID: `81212a96-09ed-4996-9471-8d729c8e69ed` (Firmware Error Record Reference)
- ErrorDataLength: `0x1c20` (7200 bytes)
- Payload signature consistent with memory training / DRAM register snapshot (per-channel signal eye margin grids, channel masks, SMI handler `de ad be ef` padding)

Kernel skipped parsing because of duplicate ACPI BERT tables (Intel EDK2 + AMI) — known Linux quirk on Supermicro/AMI boards. Decoded raw payload supports memory-subsystem fault attribution.

## Resolution requested

Warranty replacement of the faulty module under Micron's lifetime ECC UDIMM warranty.

-------------------
The DIMM exhibits a deterministic, reproducible hardware fault under

------------------------
1. With both DIMMs populated (this module + companion module
   S/N 802C0625024D5A02A7, same part number), MemTest86 hangs at

   MemTest86 hangs at Test 1, 70%, address region ~0x87F80D000.
   Reseating allows POST but the hang under stress remains.

3. With only this suspect DIMM installed in slot DIMMB2 (Channel 1):
   Intermittent memory training failure. When POST does complete,
   MemTest86 reproduces the identical hang at Test 1, 70%,
   address region ~0x87F80D000.

4. Control test: companion DIMM (same part, S/N 802C0625024D5A02A7)
   installed alone in the same system passes 4 full MemTest86 passes
   (all 13 tests including Hammer test), 0 errors.

This rules out motherboard slot, CPU IMC, BIOS, and companion DIMM as
the fault source. The failure follows the suspect module across slots.

Production impact

with FATAL severity on the subsequent boot:

  BERT: [Hardware Error]: Skipped 1 error records
  BERT: Total records found: 1

The CPER payload signature is consistent with a memory subsystem fault.

## Follow-up actions (post-RMA)

- Until replacement arrives: host runs single-channel 32GB on DIMMB2 (good module) in slot DIMMA2 (board's preferred single-DIMM slot).
- After replacement: install replacement in DIMMB2, restore dual-channel 64GB.
- Schedule BIOS update to Supermicro v5.2 (file `BIOS_X13SAEF-1C48_20251208_5.2_STDsp.zip`, dated 2025-12-08) — brings duplicate-BERT-table fix so future hardware errors get properly parsed instead of skipped. BIOS update path requires BMC iKVM virtual-media boot (SFT-OOB-LIC license not present on this BMC).
- Add EDAC monitoring: cron or Prometheus exporter on `/sys/devices/system/edac/mc/mc*/{ce_count,ue_count}` for early-warning of future corrected errors.
